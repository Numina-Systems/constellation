## Human Verification

| AC | Description | Justification | Verification Approach |
|----|-------------|---------------|----------------------|
| context-overflow-guard.AC3.1 | Agent never calls `model.complete()` with estimated tokens exceeding `modelMaxTokens` | Full end-to-end verification requires a running LLM, realistic tool definitions, and multi-turn conversation history that naturally approaches the context limit. The automated integration test in `agent.test.ts` covers the structural wiring with mocks, but cannot reproduce real-world token estimation drift where the `length/4` heuristic diverges significantly from actual tokenisation. | Run the agent locally with a small-context model (e.g., a 4K or 8K context Ollama model). Feed it a long conversation that approaches the limit. Observe logs for "pre-flight guard" warning messages. Confirm no `400`/`413` errors from the model provider indicating an oversized request was sent. Repeat with compaction intentionally disabled (set `context_budget` very low) to force the pre-flight guard to fire. |
| context-overflow-guard.AC2.1 | Compaction retries on timeout error with exponential backoff | Backoff *timing* is difficult to assert precisely in unit tests without mocking the clock or accepting flaky timing assertions. The unit test verifies retry count and chunk reduction. Actual wall-clock backoff correctness (1s, 2s, 4s delays) is best confirmed by observation. | Run compaction against a slow local model (e.g., Ollama with a large model on constrained hardware). Set `compaction_timeout` to a very short value (e.g., 5000ms) to trigger timeouts. Observe logs for retry attempts with increasing delays. Confirm chunk size reduction in log output. |
